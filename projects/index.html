  • Maps and Traffic

    A Naive Implementation of Google Maps

    The final pair project for "CS032: Intro to Software Engineering", we were required to amalgamate our first three projects to create an interactive map GUI which met the following specifications:

    • A birdâ€™s eye view of the streets that can be zoomed and panned using mouse scrolling/dragging
    • Some way to input intersections by typing in the names of streets using autocompletion
    • Clicking on the map to input the nearest intersection/point
    • Displaying the shortest path by highlighting the relevant street segments on the map screen
    • Some way to clear the shortest path
    • A starting location on Brown's campus
    • Update map with traffic information in real time by coloring ways with traffic data using your coloring scheme
    • Update A* search to account for traffic levels by scaling the weight of an edge by its traffic

    My partner and I split the work into frontend and backend; I implemented the backend in its entirety with the following features:

    • Extensive processing and efficient querying of large data (database of graph of Rhode Island)
    • A* search with latitudes and longitudes while accounting for curvature of the Earth
    • KD-Searching for nearest neighbor to a clicked point
    • Trie backed suggestions engine with smart ranking supporting whitespace edit, prefix matching, and levenshtein edit distances
    • Effective networking communications through efficient data structures and batching requests
    • Threaded the entire backend using and/or creating thread-safe data structures

    Unfortunately, the repo cannot be made publicly available due to class policies. Please feel free to contact me and I will gladly share the repo with your organization. Buuuuuut if you'd like to try it out, I have it hosted in the link below! The performance will be slow due to a low quality server (which is why it's served over HTTP). Please just refresh the page if it freezes.

  • Brewer

    Scratch Inspired Block Programming for Java

    The final group project for "CS032: Intro to Software Engineering", our group of four collaborated and pitched the idea of a Java version of Scratch. United in our love for coffee and the Java programming language, we decided to name the project "Brewer".

    Once again, the project was broken up between frontend and backend. Raphael and I worked on the backend while Noah and Ben worked on the frontend. I personally contributed the following:

    • Helped implement program parsing and type checking
    • Created database communications for program storage and loading
    • Wrote the entire test suite for network communications and program evaluations

    Currently, Ben has the public repo on GitHub: https://github.com/benjaminmurphy/CS32Brewer


    Build something simple, or Skynet (challenge accepted?). Save your program, reload it. Don't forget to hit the kill button if you find your program in an infinite loop (unfortunately, we were unable to solve the Halting Problem in time for the project deadline).

  • My Website

    Full Stack LAMP Site

    Ongoing Project

    This is a digital home I'm building for myself within the Internet Of Things. Currently, it is being born as a two week project that I'm starting in my two weeks before school. I hope to continually progress it as time passes and forge it into a digital hub that I can truly be proud of.


    Before building this site, I had minimal exposure and knowledge of the following:

    • HTML/CSS
    • Responsive Design
    • Interactive Design
    • Javascript/AJAX/jQuery
    • General Networking (Proxies, Firewalls, Hostnames, etc.)
    • Basic Bash Scripting
    • Bash Scripting Task Automation
    • Linux
    • Apache
    • SSL Protocol

    Due to this project, I am now fairly proficient in the above; I will continue building this site and look forward to incoporating the following in hopes of achieving proficiency in such topics:

    • Network Security and Penetration Testing
    • Scalable Databases and Storage
    • Website Scalability and Automation
    • PHP
    • Ruby on Rails
    • Machine Learning
    • Beautiful HTML/CSS/JS/PHP Code (half joking, 4/5's serious)

    As you can see, I hope to continually move topics from the second list to the first list while steadily adding to the second list. In such a sense, this will be a dynamic page of results, reflecting my experience and learning from creating this site.

    On the off chance that you're actually visiting this distinct page and not my actual site, I highly encourage you to check out my site with the link below...heck, even if you're on my site click the link and explore it again :). Thanks for checking out what you have thus far.
